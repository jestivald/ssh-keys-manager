# 🔑 SSH Keys Manager

Простой PowerShell-скрипт для управления SSH-ключами нод RemnaWave (и не только) на Windows 11.

Каждый ключ лежит в своей именованной папке, удобно когда нод много.

## ⚡ Быстрый старт (одна команда, без скачивания)

Открой PowerShell на любом ПК и вставь:

```powershell
irm https://raw.githubusercontent.com/jestivald/ssh-keys-manager/main/ssh-keys.ps1 | iex
```

Скрипт запустится прямо из памяти, на диск ничего не сохраняется. Работает с любого компа, где есть интернет.

## ✨ Возможности

- 🔑 Генерация Ed25519 ключей (по одному или пачкой)
- 📂 Автоматическое создание папок с ключами
- 📋 Публичный ключ сразу в буфере обмена (Ctrl+V и готово)
- 📑 Список ВСЕХ ключей (любые `.pub` файлы, в корне и в подпапках)
- 🏷️ Автоопределение типа ключа (ed25519, rsa, ecdsa, dsa)
- 🗑️ Умное удаление (папку целиком или только файлы ключа)
- 💾 Запоминает путь к папке между запусками
- 🎨 Красивый интерактивный интерфейс

## 📦 Структура файлов

После генерации получится такая структура:

```
ssh-keys/
├── ru-node/
│   ├── ru-node          (приватный ключ)
│   └── ru-node.pub      (публичный ключ)
├── de-node/
│   ├── de-node
│   └── de-node.pub
└── us-node/
    ├── us-node
    └── us-node.pub
```

Но скрипт найдёт и покажет ключи с любыми именами и в любых подпапках, не только такие.

## 🚀 Способы установки

### Способ 1: Запуск с GitHub (рекомендую)
Ничего не скачиваешь, просто запускаешь одной командой:
```powershell
irm https://raw.githubusercontent.com/jestivald/ssh-keys-manager/main/ssh-keys.ps1 | iex
```

### Способ 2: Скачать скрипт локально
```powershell
irm https://raw.githubusercontent.com/jestivald/ssh-keys-manager/main/ssh-keys.ps1 -OutFile ssh-keys.ps1
.\ssh-keys.ps1
```

### Способ 3: Клонировать репозиторий
```powershell
git clone https://github.com/jestivald/ssh-keys-manager.git
cd ssh-keys-manager
.\ssh-keys.ps1
```

### Разрешение на запуск скриптов (если ещё не настроено)
PowerShell по умолчанию блокирует запуск локальных скриптов. Выполни один раз от **админа**:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Для запуска через `irm | iex` это НЕ нужно, там скрипт выполняется в памяти.

## 🎯 Использование

### Интерактивное меню (проще всего)
```powershell
.\ssh-keys.ps1
```

Откроется меню:
```
  +---------------------------------------+
  |     SSH Keys Manager (RemnaWave)      |
  +---------------------------------------+

   [1] Создать ключ(и)
   [2] Список ключей
   [3] Удалить ключ
   [0] Выход
```

### Прямые команды (для локально скачанного скрипта)
```powershell
.\ssh-keys.ps1 new      # Создать ключ(и)
.\ssh-keys.ps1 list     # Список всех ключей
.\ssh-keys.ps1 remove   # Удалить ключ
```

## 💡 Алиас для запуска из любой папки

Чтобы вызывать скрипт просто командой `ssh-keys`, добавь в свой PowerShell-профиль:

```powershell
# Открываем профиль
notepad $PROFILE
```

**Если запускаешь через GitHub (без скачивания):**
```powershell
function ssh-keys {
    irm https://raw.githubusercontent.com/jestivald/ssh-keys-manager/main/ssh-keys.ps1 | iex
}
```

**Если скрипт лежит локально:**
```powershell
function ssh-keys {
    & "C:\путь\к\ssh-keys.ps1" @args
}
```

Перезапускаешь терминал, теперь из любой папки работает:
```powershell
ssh-keys        # меню
ssh-keys new    # создать (только для локальной версии)
ssh-keys list   # список (только для локальной версии)
```

## ⚙️ Требования

- Windows 10/11
- PowerShell 5.1+ (рекомендую PowerShell 7+)
- OpenSSH Client (обычно предустановлен; если нет: Settings → Apps → Optional features → OpenSSH Client)
- Интернет для запуска через `irm | iex`

## 🔒 Безопасность

- Используется алгоритм **Ed25519**: современный, быстрый, безопасный
- Ключи создаются **без passphrase** (для автоматизации). Если нужна парольная защита, открой скрипт и измени параметр `-N '""'` на `-N`
- Приватные ключи (файлы без `.pub`) **никогда никому не передавай**
- На сервер RemnaWave вставляется только содержимое файла `.pub`
- Скрипт с GitHub запускается в памяти, но можешь всегда посмотреть код перед запуском: просто открой ссылку в браузере

## 🐛 Известные особенности

- В проводнике Windows по умолчанию скрыты расширения файлов. Чтобы видеть где `.pub`, а где приватный ключ, включи: **View → Show → File name extensions**
- Если у ключа нет приватной части (только `.pub`), скрипт пометит его `[только публичный!]` красным в списке
